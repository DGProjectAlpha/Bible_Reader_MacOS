FEATURE 1: Import modules, read, navigate, split panes (up to 8), resize
=============================================
[PASS] Import modules — ModuleManager discovers .brbmod from bundle + ~/Library/Application Support/
[PASS] Read verses — DatabaseService.fetchVerses() async, displayed in ReaderView
[PASS] Forward/back chapters — BibleStore.navigateChapter() with book boundary wrapping
[PASS] Split horizontal — ReadingPane.splitDirection = .horizontal, ReaderArea uses HSplitView
[PASS] Split vertical — ReadingPane.splitDirection = .vertical, ReaderArea uses VSplitView
[PASS] Resizing — HSplitView/VSplitView are natively resizable
[ISSUE] Max pane limit — BibleStore.addPane() has NO upper bound check.
        Code allows unlimited panes. Should cap at 8 per your requirement.
        Fix: Add guard panes.count < 8 in addPane()
[ISSUE] No file picker for user module import — ModuleManager only discovers modules
        already in the app support directory. SettingsView shows the folder path but
        there is no "Import Module" button with a file picker (.fileImporter) to let
        the user browse and copy a .brbmod file into the modules folder.
        Fix: Add .fileImporter modifier to SettingsView with copy-to-modules logic

FEATURE 5: Search in any language (English/Russian)
=============================================
[PASS] Search — UIStateStore.performSearch() queries DatabaseService.searchVerses()
[PASS] SQL query — Uses LIKE %query% which works for any Unicode text including Cyrillic
[PASS] SearchView — text field, results list, navigate on tap
[ISSUE] Search scope — Currently searches only the active module (first pane's module).
        No module picker in SearchView to choose which module to search.
        Russian users would need RST loaded as active module to search in Russian.
        Fix: Add module selector dropdown in SearchView, or search across all loaded modules

=============================================
FEATURE 6: Sync bible panels — scroll together, navigate together from strongs/refs
=============================================
[ISSUE — MISSING] Panel sync / linked scrolling is NOT implemented.
        BibleStore has no sync mechanism. Each pane scrolls independently.
        No "link panes" toggle exists. No shared ScrollViewReader position.
        This is a significant missing feature.
        Required implementation:
        1. Add syncEnabled: Bool to BibleStore
        2. Add syncedLocation: BibleLocation that all synced panes follow
        3. When one pane navigates, update syncedLocation
        4. Each ReaderView watches syncedLocation and scrolls to match
        5. Use ScrollViewReader + scrollTo() for verse-level sync
        6. Toggle button in toolbar to enable/disable sync
[ISSUE — PARTIAL] Navigate together from strongs/refs — InspectorView navigates
        the active pane only. With sync enabled, it should navigate all synced panes
        to the equivalent verse (versification-aware).

=============================================
FEATURE 7: Settings — import modules, change localization (English/Russian)
=============================================
[PASS] Settings window — SettingsView with font size, module list, about section
[PASS] Module list — shows bundled vs user modules with paths
[ISSUE — MISSING] Module import UI — No file picker / .fileImporter to import .brbmod files.
        User must manually copy files to the modules folder.
        Fix: Add "Import Module" button with .fileImporter(forType: .data)
[ISSUE — MISSING] Localization — No language switching. All UI strings are hardcoded English.
        No .strings files, no Localizable.xcstrings, no language picker in settings.
        Required implementation:
        1. Create Localizable.xcstrings with English + Russian translations
        2. Replace all hardcoded strings with String(localized:) or NSLocalizedString
        3. Add language picker in SettingsView (or respect system locale)
        4. Book names need Russian equivalents (Бытие, Исход, etc.)

=============================================
[ISSUE — MISSING] PDF export is NOT implemented. No export UI, no PDFKit usage,

        2. Note selection UI (checkboxes, select all/none)
        3. Sort order picker (by date, by book order, custom drag)
